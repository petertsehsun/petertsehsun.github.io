About Myself

I am a PhD student in Computer Science at Queen's University in the Software Analysis and Intelligence Lab (SAIL), and I am currently working in the Performance Engineering department at BlackBerry. I obtained my Bachelor's degree in Computer Science at the University of British Columbia, and MSc from Queen's University.

My current research focus is on improving the performance of large-scale database-centric software systems. I am particularly interested in applying Program Analysis and Data Mining techniques to help improve system performance. The bug detection tools that I designed and developed are deployed in production systems. The tools are able to detect different types of performance problems. For example, in one of the use cases, the tool helped improve memory usage on the database server by over 70%. The tools are now in the process of patent application (see my ICSE 2014 paper).

I also have research experiences and interests in many areas of Software Engineering and Data Mining, including text analysis, mining software repositories, and empirical software engineering.

Projects

  • DAL Checker. DAL Checker is a static analysis tool for detecting data-access-layer related problems. The tool is designed and implemented by my, and is integrated to the BlackBerry build system. The tool is executed on a daily basis to detect new problems. So far the tool has discovered many performance-critical bugs, and has helped improve system memory performance by over 70%.

  • Tuning Machine. Tuning Machine is an automated system configuration tunning tool for customized workloads. Tuning Machine uses Kafka and Spark to do real time monitoring of servers, and learn system behaviours using Machine Learning (ML) models. Then, Tuning Machine uses the ML models to find the optimal server configuration to improve system throughput.
